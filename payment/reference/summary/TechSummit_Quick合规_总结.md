## 整体概要

这份 PPT 是一场 AWS 金融行业架构师做的 TechSummit 演示，主线是「合规三条战线同时开火、人力线性追不上监管指数增长」这一痛点，引出用 **Amazon Quick**（AWS 把 QuickSight BI + Q Business 企业 AI 合并升级后的生成式 AI 工作平台）来重塑跨境支付合规工作流。全篇结构为「痛点（PAIN POINTS）→ 产品（PRODUCT）→ 现场演示（LIVE DEMO）→ 方法论（TAKEAWAYS）」四章。

核心论点：跨境出海支付公司面临**双重监管**（境内外管局 SAFE 盯资金真实性 + 境外 MAS/FinCEN 各有规矩），合规要同时扛三条战线——**多辖区监管变动**（新规一周一变、靠人读人比对改 SOP）、**商户准入 KYB**（查制裁/穿透 UBO/核店铺真实性，一单几小时）、**反洗钱筛查 AML**（海量流水靠人眼捞可疑集群）。痛的本质是「人力线性、监管指数」，现状工具三大短板：知识散落（SOP 在 Word、原文在 PDF、记录在 Excel）、BI 太被动（只展示已知、不主动发现风险）、通用 AI 不可溯源（合规审计致命）。

Amazon Quick 的对症点提炼为三条且现场逐一兑现：①**可溯源**——问监管/SOP 带原文出处（Quick Index + Spaces）；②**Agent**——差距与风险自动算（Agents + Flows）；③**Dashboard**——风险态势实时看（Quick Sight）。关键差异化是「可溯源 + 非技术合规人员零代码可用（自然语言建看板/建 Agent）」，并提供 Web + Desktop 双入口（敏感文件 local-first）。

现场用虚构公司 **XPay** 的脱敏跨境数据，围绕 MAS Circular PSO-N02-2026（演示设定：7/1 生效）走完一条完整合规生命周期：**知识库可溯源问答 → SOP 差距比对 Agent → KYB 准入评分 Agent → AML 告警调查 + 跨辖区 SAR/STR 起草 Agent → 合规驾驶舱 Dashboard**，全程贯穿 human-in-the-loop（AI 给评分+依据，人做最终决策）。方法论收尾为「四步把合规搬上 Quick」（沉淀知识 → 配 Agent → 建 Dashboard → 闭环人审），并以 Agent/Dashboard/方法论模板留资。

## 第3页：章节扉页 — 01 PAIN POINTS（痛点）

## 第4页：合规人的一天——三条战线同时开火

- **多辖区监管变动**：新规一周一变、原文几十页，靠人读、人比对、人改 SOP。
- **商户准入 KYB**：查制裁 / 穿透 UBO / 核店铺真实性，一单几小时。
- **反洗钱筛查 AML**：海量交易流水，可疑集群靠人眼捞，漏一个就罚单。
- **核心痛点**：任何一条都能压垮一个团队，而合规要同时扛三条。
- ⚠️ **双重监管**：出海公司境内外管局（SAFE）盯资金真实性，境外 MAS/FinCEN 各有规矩，一笔交易两头交代。

## 第5页：合规人的一天——事情永远干不完

- 用一天时间轴铺排 7 件并行的具体场景：
  - 9:00 47 条交易告警逐条研判（漏一条就罚单）
  - 10:00 BD 催商户入网（制裁/UBO/店铺，几小时一单）
  - 11:00 MAS 新规将生效（不到两周，几十页要人读）
  - 13:00 可疑→起草 SAR（中+新两地，格式都要交代）
  - 14:00 付款制裁命中（同名误报一堆，人工逐个清）
  - 15:00 复审/牌照到期（没人盯就漏）
  - 17:00 要风险日报（数据散落，靠人汇总）
- 结论：七件事没有一件能拖到明天，而这只是一个人的一天。

## 第6页：痛的本质——人力线性，监管指数

- 图示：监管/商户/交易量呈指数 ▲ 上升，合规人力几乎不变（线性）。
- 三个量化锚点（演示设定）：MAS 新规 PSO-N02 不到两周生效；单条 SOP 差距潜在最高罚款 **S$5,000,000**；一个高风险商户人工背调需数小时。

## 第7页：现状工具为什么不够

- **知识散落**：SOP 在 Word、原文在 PDF、记录在 Excel，问一个问题翻三个系统。
- **BI 太被动**：传统 BI 只"展示已知"，不主动发现风险。
- **AI 不可溯源**：通用 AI 答了却拿不出依据——合规审计时这是致命的。

## 第8页：合规要的是什么

- ① 问监管/SOP **要带出处**（可溯源）；② 差距与风险**要自动算**（Agent）；③ 风险态势**要实时看**（Dashboard）。
- Amazon Quick 一一兑现。

## 第9页：章节扉页 — 02 PRODUCT（产品：Amazon Quick 是什么）

## 第10页：Amazon Quick 是什么

- **大白话比喻**：像给合规团队配了一个助手——读过你所有监管原文和 SOP，会自己查、自己算、还能搭看板，用大白话提问/下指令，直接给出带出处的答案、报告和报表。
- **一句话定义**：AWS 的生成式 AI 工作平台，由 **QuickSight（BI）+ Q Business（企业 AI）合并升级**而来。
- **六大能力**：Sight · Flows · Automate · Index · Research · Apps。

## 第11页：Quick 的四件套

- **Chat（聊）**：用大白话提问、下指令。
- **Agents（干活）**：自动查、比对、评分、起草。
- **Spaces（聚资源）**：监管原文 + SOP 进知识库。
- **Integrations（连系统）**：接现有数据与工具。
- 闭环：Chat 提问 → Agents 干活 → Spaces 供知识 → Integrations 连系统。

## 第12页：Quick 怎么对症合规三痛

| 痛点 | Quick 能力 | 怎么解 |
|---|---|---|
| 监管/SOP 散、不可溯源 | Quick Index + Spaces | 原文+SOP 入库，问答带引用来源 |
| 差距/风险靠人算 | Agents + Flows | Agent 比对新规 vs SOP、给商户评分 |
| 风险看不见 | Quick Sight Dashboard | KPI 实时预警、到期日历、AML 告警 |

## 第13页：关键差异——可溯源 + 非技术人员可用

- **可溯源**：每条 AI 回答都附原文出处——合规审计刚需（AI 说了，依据拿得出）。
- **非技术可用**：自然语言建看板、建 Agent，不用写代码，合规同事自己上手。
- **双入口**：Web（办公室）+ Desktop（本地敏感文件，local-first）。

## 第14页：今天演给你看的全链路

- 一条完整合规生命周期：**读 → 落 → 挡门 → 持续盯 → 看态势**
  - ① 知识库（可溯源问答）② 差距 Agent（新规 vs SOP）③ KYB（准入风险评分）④ 交易监控（告警+跨辖区上报）⑤ Dashboard（实时预警）

## 第16页：演示场景设定

- ⚠️ **免责声明**：XPay 为虚构公司，全部脱敏模拟数据，与任何真实公司无关。
- 背景：XPay 跨境支付合规团队；新规事件：**MAS Circular PSO-N02-2026**（已发布、7/1 生效、距今不到两周）；数据为跨境场景脱敏模拟。

## 第17页：战线一 · 监管知识库（可溯源问答）约 3min

- 原工作流：新规几十页人读 + 人比对现行 SOP，问一个点翻原文半天。
- **录屏1**：MAS 新规 + 现行 SOP v3.4 入 Space → 问"新规对 KYB 有哪些新增要求？"→ 回答带原文引用，点开直达原文段落（"可溯源"的兑现）。

## 第18页：战线一延伸 · Agent 自动差距比对 约 2.5min

- 原工作流：新规 vs 现行 SOP 靠人一条条比对，两天才出一份差距报告。
- **录屏2**：**SOP 合规审查官 Agent** 出《差距分析报告》——5 处差距（3 硬性缺口 / 2 操作细化）、风险敞口（检查命中概率极高、潜在罚款 S$5M）、Quick Flows 周一定时自动跑 + 推审批卡片。
- 要点：人要两天，Agent 几分钟，还自动定时复查。

## 第19页：战线二 · KYB 商户准入 Agent 约 3min

- 原工作流：一个商户查制裁/UBO 穿透/店铺真实性，人工背调几小时，BD 天天催。
- **录屏3**：**KYB Onboarding Officer Agent** 处理"星辰跨境"申请——72 秒出报告，综合风险评分 **72/100（高风险）**；World-Check 命中 2 条 / 2 未披露 UBO；店铺真实性 0/4；MAS 辖区不符；建议拒绝或转 EDD 待复核（human-in-the-loop）。
- 衔接：AI 给评分+依据，人做最终决策——但已入网商户每天还在刷告警。

## 第20页：战线三 · 交易监控告警调查 + SAR 起草 约 3min

- 原工作流：已入网商户每天刷海量告警，人逐条研判一条几十分钟，漏一条就是罚单。
- **录屏4**：**AML 告警调查 Agent** 处理一条告警——"鸿运跨境"单周回款突增 **480%**、资金 T+0 快进快出；Agent 拉历史交易+KYB+关联店铺综合研判；结论：疑似借道结汇/贸易背景不真实，涉 3 店铺；**跨辖区命中：中国 SAFE + 新加坡 MAS/STRO 双报告**；人确认 → Agent 自动起草 STR/SAR 上报草稿。
- 要点：几分钟出研判+依据+上报草稿，多辖区义务一次看清。

## 第21页：报表 · 合规驾驶舱 约 2.5min

- 原工作流：风险态势靠人到处汇总日报，数据散在各系统。
- **录屏5**：**合规 KPI Dashboard 实时看板**——进审量/机审率/SLA、辖区事件分布、AML 告警、牌照 90 天到期预警。

## 第22页：全链路走通

- ①知识库 ②差距 Agent ③KYB ④交易监控 ⑤报表 全部打通：挡门 → 持续监控 → 发现可疑 → 跨辖区上报，一条完整合规生命周期跑通；非技术合规同事也能复用。

## 第24页：方法论——四步把合规搬上 Quick

1. **沉淀知识**：监管原文 + SOP → Space 知识库。
2. **配 Agent**：差距比对 / 风险评分 / 定时复查。
3. **建 Dashboard**：KPI + 预警。
4. **闭环**：human-in-the-loop 复核 + 审批。
