# 公司画像模板（所有玩家文件统一遵循）

> 本文件定义 `02c-epayment-players/` 与 `03c-crossborder-players/` 下每家公司文件的统一结构。
> **写作原则**：① 第一性——先一句话定位"它是产业链哪个角色"，再展开；② 可信度严格标注（见下）；③ 硬事实（股东/牌照/估值/营收/规模）必须 deep-research 核查带引用，不凭记忆；④ 每项标数据年份，避免旧数据当现状。

## 可信度图例（每条事实句尾标注）
- 📌 **已核查·一手**：经 deep-research 多源对抗式核查、引用权威一手来源（招股书/财报/央行牌照名录/公司官网/监管文件），句末标来源 + 数据年份。
- 🔧 **行业公知/推理**：成熟机制或分析推理，教学性，未逐条引用。
- ⚠️ **未核实/时效**：凭印象、待核查、或会随时间漂移的状态——明确标出，绝不伪装成事实。

## 完成度分级
- **头部（必做）**：15 维写全。
- **腰部（扩展）**：写核心 6-8 维（定位/背景/股东/牌照/业务/区域/护城河），其余留待补。

# 〔公司名（中文 / 英文）〕

> 📌 **一句话定位**：（产业链哪个角色 + 最关键标签）
> ⚠️ **数据时效**：本文事实截至 〔YYYY-MM〕；支付公司动态变化快，引用前核对最新。

## 1. 基本信息
- 成立年份 / 总部 / 创始人 / 当前状态（上市代码·未上市·被收购）

## 2. 背景与沿革
- 怎么起家、关键转折点（业务转型/牌照获取/被收购/上市）

## 3. 股东与资本 ⚠️强需核查
- 主要股东 / 历轮融资·估值 / 上市情况（📌 标来源+年份）

## 4. 牌照与资质 ⚠️支付公司命脉
- 央行支付牌照（业务类型）/ 各国收单·跨境·储值·EMI 牌照 / PCI-DSS 等（📌 标来源）

## 5. 定位与商业模式
- 产业链位置（网关/收单/聚合/钱包/跨境收款/清算网络）
- 怎么赚钱（费率/take rate/金融增值/服务费）——呼应 `02-epayment-business §6`

## 6. 核心产品与业务范围
- 主要产品线、覆盖的支付方式/场景

## 7. 业务区域
- 覆盖市场/地理、主战场 vs 拓展区

## 8. 规模与数据 ⚠️标年份
- TPV / 营收 / 商户数 / 用户数（📌 标来源+年份；无公开则标"未披露"）

## 9. 组织架构
- 集团结构、主要运营主体、关联实体（如蚂蚁↔支付宝↔网商银行）

## 10. 技术架构特点（若公开可得）
- 自建/上云、特色技术能力、是否 AWS/其他云客户——服务"聊技术架构"目标

## 11. 护城河与差异化
- 凭什么守得住（呼应 `02-epayment-business §7/§7.1`）

## 12. 主要竞争对手
- 直接对标、所处竞争格局

## 13. 最新动态与风险 ⚠️时效
- 监管/合规/近期重大变化

## 14. 与本研究 / AWS 对话的衔接
- 这家适合聊什么业务/技术话题、AWS 能切什么（SA 视角）

## 15. 来源清单
- 本文 📌 已核查项的一手来源链接（按维度列出）
